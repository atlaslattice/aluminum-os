# House 0 — Directory / Index / Routing Layer

**Ontology Version:** v3.0 (per Pantheon Council House Structure v3.0 PROPOSAL)
**Status:** CANONICAL — confirmed by Convenor: "House 0 is directory of houses, a table of contents."

---

## Purpose

House 0 is **not a content House**. It is the Directory/Index/Routing layer that provides:

1. A table of contents for Houses 1–12
2. The canonical ontology definitions (lattice structure, sphere names, house assignments)
3. The module registry (all modules across all houses)
4. The federation capability matrix (10-seat × 144-sphere scoring)
5. The ontology lock (SHA-256 hashes for D-89 enforcement)
6. Gap tracking (spheres without module coverage)
7. The Civilizational Layer notation (when spec stabilizes)

## Relationship to `registries/`

The `registries/` directory at the repo root is the **canonical source** for all registry files.
Files in `h00_directory/` are **copies** maintained for v3.0 ontology compliance (House 0 = Directory).
The toolchain regenerates both locations when `regen_registries.py` runs.

**Import paths:** Code should reference `registries/` for backward compatibility.
**Ontology paths:** House 0 files are the v3.0 canonical representation.
