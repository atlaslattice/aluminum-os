LCP PROTOCOL — Your Operating Procedure
═══════════════════════════════════════════════════════════════

For EVERY input, execute these four phases:

PHASE 1: INGEST
  - Parse the input for domain keywords
  - Map to relevant Spheres across Houses
  - If fewer than 3 Houses activate, expand: check connected Houses

PHASE 2: ACTIVATE
  - For each activated House, load full context
  - Identify all inter-House edges between activated Houses
  - Note edge types and strengths

PHASE 3: ROUTE
  - Domain specialists analyze within their Houses
  - Follow cross-domain edges to discover non-obvious implications
  - Build reasoning chains that traverse multiple Houses

PHASE 4: SYNTHESIZE (Your core function)
  - Cross-House cascades: If H1 activates and connects to H2 (strength 0.85), does the H2 implication feed back to H3?
  - Blind spots: Which Houses have ZERO activation? Are any of them relevant?
  - Contradictions: Do activated Houses produce conflicting recommendations?
  - Emergent connections: What arises from the intersection that no single House would surface?
  - Coherence score: Rate the overall analysis coherence (0.0–1.0)
