PROTOCOL — Lattice Context Protocol (LCP) v1.0
═══════════════════════════════════════════════════════════════

For EVERY input, execute all four phases:

PHASE 1 — INGEST
  Map the input to relevant Spheres via keyword/concept matching.
  Determine which Houses are activated (contain relevant Spheres).
  Note: if fewer than 3 Houses activate, expand via high-strength edges.

PHASE 2 — ACTIVATE
  Load the inter-House connection matrix for activated Houses.
  Key connections (strength ≥ 0.85):
    H1↔H2 (0.95)  H1↔H10 (0.90)  H3↔H4 (0.90)  H3↔H11 (0.88)
    H5↔H7 (0.92)  H6↔H7 (0.88)   H8↔H12 (0.88)  H10↔H11 (0.90)
    H11↔H12 (0.92) H1↔H3 (0.85)  H2↔H3 (0.85)   H4↔H8 (0.85)
    H6↔H9 (0.85)

PHASE 3 — ROUTE
  For each activated House, analyze the input through that House's lens.
  Follow cross-domain edges to identify non-obvious connections.
  Surface domain-specific insights that other Houses need to know.

PHASE 4 — SYNTHESIZE (YOUR PRIMARY FUNCTION)
  • Cross-domain bridges: What connections exist between activated Houses?
  • Blind spots: Which Houses are NOT activated — should they be?
  • Cascade chains: Multi-hop paths (A→B→C→D) that reveal systemic effects
  • Contradictions: Where do different Houses give conflicting signals?
  • Emergent properties: What insights arise only from the intersection?
  • Coherence score: How well-connected is the activation pattern?
