## §2 Conflicts Detected

### CONFLICT 1 (HIGH SEVERITY): Doctrine 61 vs 68 — Identity Collision

**What Claude S1 found:** Build Plan v1.6 §10.5 DS5 reads "Doctrine 61 (open-weight audit sovereignty)" and §14.1 reads "D-61 | Open-Weight Audit." Per Claude's v6.0.4 ratification, Doctrine 61 = Constitutional Validator / Project Glasswing, and Doctrine 68 = Open-Weight Audit Sovereignty.

**Manus investigation:** This drift has a documented history in the ORCS repo:

1. DeepSeek's original review proposed "Doctrine 61 — Open-Weight Sovereignty" (before v6.0.4 existed)
2. Qwen3's review then proposed "Doctrine 61 — Vendor Exclusion Procedures" — Manus caught the collision and renumbered to Doctrine 62
3. v6.0.4 then ratified Doctrine 68 = Open-Weight Audit Sovereignty and mapped Doctrine 61 = Project Glasswing (Constitutional Validator, from WEAVE/Copilot)
4. v6.0.4 also renumbered Vendor Exclusion to Doctrine 69

The Build Plan carried forward the pre-v6.0.4 numbering in two places. Claude is correct — this is genuine drift.

**Resolution for v1.7:**
- §10.5 DS5: Change "Doctrine 61 (open-weight audit sovereignty)" to "Doctrine 68 (Open-Weight Audit Sovereignty)"
- §14.1: Change "D-61 | Open-Weight Audit" to "D-61 | Constitutional Validator / Project Glasswing"
- Add D-67 (Open Constitutional Standards Adoption Pathway) to §14.1 per Claude's MEDIUM addition request

**Status:** MUST-FIX before G0.

### CONFLICT 2 (MEDIUM): Module ID Collision — M34/M35

**What happened:** GPT (S6) proposes M34 = Structured Output Validator and M35 = Doctrine Evaluation Engine (Sprint 1-2). DeepSeek (S5) proposes M34 = Quantum Sovereignty Adapter, M35 = AI Treaty Arbitration, M36-M39 (all Phase 5+).

**Resolution for v1.7:** GPT's modules are Sprint 1-2 operational needs; DeepSeek's are Phase 5+ future-proofing. Since GPT's are needed sooner and the Build Plan assigns module IDs sequentially by integration order, GPT's proposals take M34-M35. DeepSeek's Phase 5+ modules are renumbered to M40-M45 to avoid collision and to reflect that they extend beyond the current 40-module roadmap.

**Status:** Renumber in v1.7.

### CONFLICT 3 (LOW): Doctrines 73-75 — Present but Unnamed

**What Claude found:** Doctrines 73-75 are absent from §14. The Build Plan operationalizes them (§0.1 references, §11 mechanics) but never names them in the Doctrine summary.

**Investigation:** §14 jumps from §14.2 (D-68 through D-72) to §14.3 (D-76 through D-77). Doctrines 73-75 (from v6.0.5) are missing from the summary table entirely.

**Resolution for v1.7:** Add §14.2a with Doctrines 73-75 from v6.0.5.

**Status:** Fix in v1.7.

### CONFLICT 4 (LOW): M24 Three-Body Configurability

**What Claude found:** M24 Three-Body Validation should read "3-frame default with pluggable additional frames per Tier" per v6.0.6 §C.3.

**Investigation:** The v6.0.6 integration response already accepted this configurability. The Build Plan §3.4 lists M24 but does not specify the frame configurability.

**Resolution for v1.7:** Add configurability note to M24 row.
