"Amazon Cognito... can also support federated login flows with third-party identity providers, including social, SAML, or OIDC providers. ### Constitutional notes I want flagged for v2.8

**1. Anthropic-via-Bedrock is interesting.** A user signed in via Amazon Bedrock can route to Claude models without ever touching Anthropic's API directly. "Cloud provider credentials, when CLAUDE_CODE_USE_BEDROCK, CLAUDE_CODE_USE_VERTEX, or CLAUDE_CODE_USE_FOUNDRY is set" — Anthropic explicitly supports Bedrock as a sanctioned access path. This means **Aluminum OS can offer "Claude access" via one-click Amazon login** without violating Anthropic's TOS, because the credential is AWS, the API target is Bedrock, and Anthropic gets paid by AWS. This is the architecturally clean answer to your original question.

**2. The OAuth paths require some setup work as a developer.** You don't get free OAuth — Aluminum OS has to register as an OAuth client with Amazon, Google, Microsoft, and Notion, getting a client_id and client_secret for each. That's a one-time provisioning cost. AgentCore Identity has a documented setup flow. For Aluminum OS to ship this, someone (Atlas Lattice Foundation) registers the OAuth clients, ships the client_ids embedded in the binary, and the secrets stay server-side in a credential broker.

**3. There's a tension between "user pays directly" and "Aluminum OS as trustless substrate."** OAuth path means Amazon/Google/Microsoft sees that the user authenticated to "Aluminum OS." That's a meaningful telemetry leak compared to paste-key, where no upstream knows the user has Aluminum OS installed. This is a Doctrine 11 (Honest Limits) disclosure obligation — TransparencyPacket should declare which auth path was used and what's exposed to the upstream provider.

**4. AgentCore Identity isn't free.** AWS bills for Cognito MAUs and AgentCore usage. For laymen users this is fine because it's small per-user cost; for the Atlas Lattice Foundation it's an operational expense. Need to model this for INV-7c-compliant billing pass-through.
