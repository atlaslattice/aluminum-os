## §2 Novel Research Conducted

Six research threads were pursued to discover symbiosis points not yet identified by any Council seat:

### Thread 1: W3C Agent Identity Registry Protocol
The W3C launched a new Community Group on April 24, 2026 (4 days before this synthesis) specifically developing open specifications for verifiable AI agent identity infrastructure. The group's scope includes DID method specifications, agent credential formats based on W3C Verifiable Credentials, trust negotiation protocols, and integration profiles with MCP, A2A, OAuth/OIDC, and SPIFFE. An IETF Internet-Draft (draft-drake-agent-identity-registry-00, April 11, 2026) formalizes the registry as W3C VCs for decentralized identity integration. This is the exact infrastructure that M118–M125 (Switzerland Layer) requires, and it did not exist when those modules were designed.

### Thread 2: AgentCity Constitutional Governance (arXiv, April 8, 2026)
Academic researchers independently arrived at the same governance architecture as the Build Plan's Pantheon Council. AgentCity proposes a "Separation of Power" model where agents legislate rules as smart contracts, deterministic software executes within those contracts, and humans adjudicate through a complete ownership chain. The paper identifies the "Logic Monopoly" — the exact threat that INV-7 prevents — and proposes alignment-through-accountability as the solution. This is independent academic validation of the Build Plan's constitutional framework.

### Thread 3: a16z "Missing Infrastructure for AI Agents" (April 16, 2026)
a16z identifies five infrastructure gaps: identity for non-humans, governing AI-run systems, payment rails for AI-native businesses, repricing trust in an agentic economy, and preserving user control. Their "KYA (Know Your Agent)" concept maps directly to TransparencyPacket. The x402 protocol (payments embedded in HTTP headers, $1.6M/month real volume) is the natural payment rail for INV-7c-compliant billing. The paper explicitly states: "whoever controls the model ultimately controls the outcome" — validating the multi-provider sovereignty thesis.

### Thread 4: AWS Bedrock AgentCore Identity (shipping April 2026)
Amazon's AgentCore Identity is now a production service providing secure credential brokering for AI agents: SigV4 signing, OAuth 2.0 with JWT bearer tokens, API key management, and VPC egress to customer Identity Providers. AgentCore Gateway provides centralized MCP server management. This validates the M109b architecture (one-click Amazon login → multi-model access via Bedrock) as not merely theoretical but a shipping AWS product.

### Thread 5: Regenerative Carbon-Aware Compute
The tokenized carbon market is projected to grow from $5.3B (2025) to $13.4B (2033). Carbon-Aware Nomination systems already route workloads to renewable-powered nodes. Canada's Sovereign AI Compute Strategy (April 15, 2026) invests in public AI infrastructure. Community-owned compute cooperatives are emerging. The "Regenerative" in ORC can be made literal through carbon-aware inference routing that generates tokenized carbon credits as a byproduct.

### Thread 6: FIDO2 Passkeys as Universal Hardware Trust Anchor
Apple Secure Enclave, Android StrongBox, Windows Hello, and ChromeOS Titan all speak WebAuthn/FIDO2. Passkeys provide hardware-backed, biometric-gated, cross-platform identity without vendor lock-in. This is the universal hardware root that M125 (Universal Provider Credential Vault) needs — one passkey per user unlocks all provider credentials across all platforms.
